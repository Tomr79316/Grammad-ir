Text Editors

Emacs

Martin Gregory has kindly contributed an Emacs interface to An Gramadóir which is very easy to use. It shows you a split screen with the original text file on top and the errors output by An Gramadóir below. With a single keystroke, you can move through the errors in parallel, making corrections as you go. Here are his instructions (the file gramadoir.el is included in the distribution as of version 0.3):

"If you are an experienced Emacs user, you'll probably want to auto load the file and define your own keys for the two functions listed below.

  1. put gramadoir.el in your emacs site lisp directory, if you don't have one, put it it ~/lisp
  2. put the following lines in your ~/.emacs file 
       (load-file "~/lisp/gramadoir.el")
   (global-set-key "\C-cg" 'gramadoir-check-files)
   (global-set-key [f3] 'gramadoir-next-message)
  3. start Emacs
  4. To have An Gramadóir check files, press the key ^C followed by the key g.
  5. You are prompted for files to check, enter them using either relative or absolute paths, but avoid using ~ or environment variables as gramadoir.el doesn't handle these yet.
  6. A second window will open with the messages from gr.
  7. To go to the first error position in the first file, press F3. Repeated pressing of F3 takes you to each subsequent error."

Vim

I've written a script which emulates the behavior of Martin's Emacs interface within Vim. It will be included in future distributions, or you can download it directly from Vim online. Instructions:

  1. put gramadoir.vim in ~/.vim/plugin.
  2. put the following lines in your ~/.vimrc file 
    map <F3> <Gr>
map <F4> <Amach>
    or choose different hot keys if you want.
  3. open an Irish language text file in Vim.
  4. to check its grammar, press F3; a second window will open with the messages from An Gramadóir. The cursor will move to the first error (which is also highlighted in red if you are using a suitable color terminal).
  5. Each successive use of F3 moves to the next error (in both windows, in parallel).
  6. Pressing F4 exits the grammar checker and resumes normal editing.
