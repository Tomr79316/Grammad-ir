An Gramadóir:
Porting to other languages

Introduction

I would be excited to help port this software for use with languages other than Irish. Getting something simple up and running should be easy, even if your language lacks some of the basic tools for doing natural language processing. If, however, you already have things like a large word list or a part-of-speech tagger, then you should be able to begin implementing sophisticated grammatical rules immediately.

Caveat Emptor: If you've read the Details page, you know that An Gramadóir is a rule-based grammar checker that uses pattern matching to find errors. This is quite satisfactory for Irish where many of the common errors made in writing involve misuse of the initial mutations which are determined almost entirely by local context (usually, just the preceding word). It is much harder to find errors in languages like English using the same scheme, and harder still in languages having free word order.

In the instructions below, I will use "xx" for the two-letter ISO-639-1 code for your language. As of version 0.3, I've included in the distribution a trivial port to English which can be used as a template for adding new languages.

Things to do

1. Build focail-xx.bs and eile-xx.bs. These are the databases of recognized words. If you have some grammatical rules that don't require part of speech information, then in fact this step is optional. You can construct focail-xx.bs from a plain text file containing one word per line (which can be generated easily if your language has ispell support). See the focail-en.bs target in the Makefile for an example.

An intermediate possibility would be if, for instance, you want to check for erroneous constructions like the English "He done went fishing", or "He done ate his dinner" (sadly common in my part of the world). You could add a rule to rialacha-xx.in resembling:

<N>ANYTHING</N> done <VP>ANYTHING</VP>:MESSAGE

and populate focail-xx.bs with verbs in the past tense. It all depends on what kind of checks you need to do for your language. The file eile-xx.bs is a "replacement" file which contains misspelled (or, as in the case of Irish, pre-standard or dialect) words and suggested replacements. I built the file eile-en.bs in the distribution by collating the specifically American and British word lists that are distributed with ispell.

2. Rewrite "byte_to_markup_xx" in cuardach-xx.c. You can choose the grammatical codes that go into focail-xx.bs and the corresponding XML tags as you see fit; just correctly convert the former to the latter in this one function.

3. Tweak the details. If your language has unusual conventions for ends of sentences, you may need to play with the abairti module. The buffers in cuardach.c for storing words are 32 bytes long (longest Irish word: ngearr‑chlóscríobhneoireachta, 28 bytes) and may need to be increased. Finally, be sure to add a line for your language to the TEANGACHA file of the form:

xx ENCODING BOUNDARYCHARS INTERNALCHARS

where "ENCODING" is the default character encoding for texts in your language (and also the character encoding used in the rule files), e.g. "ISO88591" or "UTF8", etc.

4. Write aonchiall-xx.in. These are the rules for disambiguation. In Irish, and I would imagine in many other languages, one can get away without too many rules here. See the description of this file on the Contributing page.

5. Write rialacha-xx.in and eisceacht-xx.in. These are the grammatical rules and exceptions. Again see the Contributing page for information on how to create these files.

6. Localize message strings. Technically this is part of porting the user interface, not porting the grammar checker. I'm guessing however that you'll want to see the output in the same language as the documents that are being checked. As of version 0.3, this is done with the GNU gettext package. You can find out more from our page at the GNU Translation Project.
